ReGit - Requirement Management Tool for Google Docs

ReGit is a simple and efficient tool for managing requirements directly within Google Docs. By leveraging Google Sheets as a database, ReGit allows for seamless creation, modification, and tracking of requirements, making it ideal for collaborative projects where requirements management is critical.

Using a Google Sheet Document as a Database

To get started, create a Google Sheet document. Rename one of the sheets as DB. In the first row, insert the following column headers:

  • ID
  • Description
  • Location
  • Parents
  • Childs
  • Ver
  • State

Connecting to the Database

In your Google Docs document, create a simple table that connects the document to the database. Insert the following:

#DB https://docs.google.com/spreadsheets/d/tHeL0ngH4ashT0th3fil3/edit

Creating a Requirement

To create a requirement, insert a simple table that includes the following information:

#ID ReqID1
#Description The description of the requirement
#State Open

The table can be of any shape. If a cell begins with a hash (#), it represents a variable of the requirement. The cell to the right contains the corresponding data for the variable.

Pushing and Pulling Requirements

Once the requirement is created, use the menu items in ReGit to push or pull the requirement from the database, ensuring that the latest information is always up-to-date.
